Java does **not support multiple inheritance of classes**, but it still lets you achieve the same outcome safely using other mechanisms.

# 🧠 What is multiple inheritance?

Multiple inheritance means:

> A class inherits from more than one class

Example (NOT allowed in Java):

```java
class A { }
class B { }

class C extends A, B { } // ❌ NOT allowed
```

---

# 🚨 Why Java does NOT support it (important interview point)

Because of the **Diamond Problem**

### Example:

```java
class A {
    void show() { System.out.println("A"); }
}

class B extends A {
    void show() { System.out.println("B"); }
}

class C extends A {
    void show() { System.out.println("C"); }
}

// If this was allowed:
class D extends B, C { }
```

👉 Now calling:

```java
D obj = new D();
obj.show();
```

❓ Should it call B’s `show()` or C’s `show()`?

👉 This ambiguity is the **Diamond Problem**

---

# ✅ How Java handles it instead

Java uses **3 safe alternatives**

---

## ✅ 1. Interfaces (primary solution)

```java
interface A {
    void show();
}

interface B {
    void display();
}

class C implements A, B {
    public void show() {
        System.out.println("A method");
    }

    public void display() {
        System.out.println("B method");
    }
}
```

👉 A class can implement **multiple interfaces**

---

## ✅ 2. Default methods (Java 8+)

Interfaces can have methods with body:

```java
interface A {
    default void show() {
        System.out.println("A");
    }
}

interface B {
    default void show() {
        System.out.println("B");
    }
}
```

Now:

```java
class C implements A, B {
    public void show() {
        A.super.show(); // resolve ambiguity
    }
}
```

👉 Java forces you to resolve conflict manually

---

## ✅ 3. Composition (best design approach)

Instead of inheriting, use objects:

```java
class Engine {
    void start() { }
}

class MusicSystem {
    void play() { }
}

class Car {
    Engine engine = new Engine();
    MusicSystem music = new MusicSystem();
}
```

👉 This is preferred in real-world systems

---

# 🧠 Key concept

| Feature                           | Java Support |
| --------------------------------- | ------------ |
| Multiple class inheritance        | ❌ No         |
| Multiple interface inheritance    | ✅ Yes        |
| Multiple behavior via composition | ✅ Yes        |

---

# 🔥 Interview-level insight

Java avoids multiple inheritance of classes to:

* Prevent ambiguity
* Reduce complexity
* Improve maintainability

Instead, it promotes:

> **“Composition over Inheritance”**

---

# 🧾 Final answer (interview ready)

> Java does not support multiple inheritance with classes due to ambiguity issues like the Diamond Problem.
> However, it achieves similar functionality using multiple interfaces and composition, ensuring better design and maintainability.
